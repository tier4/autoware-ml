# Project description

I am working on a Lidar-Lidar miscalibration detection pipeline using autoware-ml.
I want to visualize different wait to project 2 pointclouds generated by 2 lidars into a 2D frame that can be used as input to a ResNet model.
This project aims to develop a simple tkinter GUI to select the data and adjust the parameters. The data is then computed and send to a Rerun visualizer.

## Implementation Steps

Format of the input files are described in ./format.md

### 1. Read inputs

    - [ ] When the application is started, a pickle file must be selected using a tkinter interface (file dialog).
    - [ ] From the pickle file, a list of possible pointcloud files is generated and provided to the user.
    - [ ] Once the user selects a pointcloud file, the possible lidar sources are then presented to the user who must select exactly 2.
        - [ ] Information about the point indexes corresponding to the 2 lidar sources is extracted from a json file.
        - [ ] The path of the json file is the path of the pickle file with the following 2 changes:
            - folder name is changed from LIDAR_CONCAT to LIDAR_CONCAT_INFO.
            - file name extension is changed from .pcd.bin to .json.

At the end of this step, the following data needs to be available:
    - path to the pointcloud data
    - lidar sources to use (name, token, transform, point indexes).

### 2. GUI (Tkinter Control Panel)

    - [ ] A Tkinter GUI is presented to the user to configure the virtual camera parameters and projection methods.
    - [ ] Virtual Camera Extrinsics: Add sliders/spinboxes for X,Y,Z (translation) and Roll, Pitch, Yaw (rotation) to position the virtual camera relative to the overlapping FOV of the two LiDARs.
    - [ ] Virtual Camera Intrinsics: Add input fields/sliders for Focal Length (fx​,fy​), Principal Point (cx​,cy​), and target Resolution (Width, Height). This resolution must match the expected input size for the ResNet model.
    - [ ] Projection Method Selector: Add Radio Buttons to toggle between:
        - Unified Global Frame (Native 3D projection)
        - Depth Map Accumulation (Z-buffer projection to dense 2D array)
        - Spherical Panoramic Projection (Azimuth/Elevation mapping)
    - [ ] Action Buttons: Add a "Reset to Defaults" button and a "Save Configuration" button to export the current intrinsics/extrinsics to a JSON file.

### 3. Data Processing & Math Logic
    - [ ] Load the raw .pcd.bin pointcloud array.
    - [ ] Filter the array using the point indexes extracted in Step 1 to isolate the data for LiDAR 1 and LiDAR 2.
    - [ ] Use the transform data from Step 1 to align both pointclouds into a unified global coordinate frame.
    - [ ] Implement a callback function that triggers whenever a Tkinter slider is adjusted. This function must:
        - Calculate the 4x4 Transformation Matrix from the Extrinsics GUI values.
        - Apply the selected Projection Method math (e.g., transforming points to the camera frame and dividing by Z for the Depth Map method).

### 4. Rerun Visualization Integration
    [ ] Initialize the Rerun viewer natively (rr.init and rr.spawn).
    [ ] Log the unified 3D pointclouds to the Rerun world space, coloring LiDAR 1 and LiDAR 2 differently (e.g., red and blue) for visual debugging.
    [ ] Log the Virtual Camera using rr.Transform3D (linked to the Tkinter extrinsics) and rr.Pinhole (linked to the Tkinter intrinsics).
    [ ] If "Depth Map" or "Spherical" is selected in the GUI, log the resulting 2D NumPy array as an rr.DepthImage so the user can see the exact 2D tensor that will be fed into the ResNet model.
